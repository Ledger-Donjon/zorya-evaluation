// All three radius2 commands panic at processor.rs:758 because radius2
// tries to unwrap an Option that is None. 
// This happens because:
// 1. radius2 expects certain program state to exist (like argv setup, memory regions, or symbols),
//    but TinyGo binaries don’t expose argv/stack in the standard way.
// 2. Passing `-a 0x22af70` or `--crash` doesn’t help because the initial execution state
//    is missing required structures.
// 3. Using `-A a` to provide a concrete argument also fails because radius2 cannot correctly
//    map it into the TinyGo program’s memory. 
//
// In short: radius2 panics due to incomplete modeling of TinyGo binaries, not because
// the binary itself is invalid.
